What is OKF?

Open Knowledge Format, v0.1

Plain files

A note is one markdown file with YAML frontmatter. No SDK, no schema server — it works in Obsidian, git, and grep alike.

Human-readable, agent-parseable

This tool writes six owned keys — type, title, description, resource, tags, timestamp — readable without tooling, parseable with any YAML parser.

Extension-safe

OKF requires consumers to tolerate and preserve unknown frontmatter keys. The sync tool only ever touches fields it owns — bring your own extensions freely.

The tagging problem — top_n vs any

    
Naive "does this keyword appear anywhere" tagging degrades badly once a corpus is even lightly topical:
    every note ends up tagged with almost every keyword. This is the bug that motivated the tool.
    The default top_n mode ranks tags by keyword-hit count per document and keeps only the
    top N that clear a minimum hit count. Drag the sliders and toggle modes against the current note.
